# 渠道与会话

Samata 支持 CLI、飞书、企微和 Telegram 多渠道入口。所有渠道共享 Agent 能力，但每条执行路径都会携带 channel 标识，用于权限判断、日志审计和会话隔离。

## 渠道模型

- **CLI**：系统管理和调试入口，可在满足身份条件时执行系统级操作。
- **飞书 / 企微 / Telegram**：Bot 用户入口，不能天然获得 CLI 的系统管理员权限。
- **Agent 绑定**：Bot app、群、用户或渠道身份可绑定到指定 Agent。

## 会话上下文

会话层负责维护当前 Agent、用户上下文、附件状态和长期 workspace 信息。渠道适配层只负责收发消息，Agent 运行逻辑保持一致。

## 设计原则

- channel 是权限判断的一部分。
- Bot 渠道只暴露被授权的 Agent 与工具。
- 会话信息应服务于当前 Agent，不跨 Agent 泄露用户上下文。
