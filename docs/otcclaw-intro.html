OTCClaw 衍语 · 背景
衍生品展业【知识沉淀】与 AI 赋能方案
团队经验丰富却分散各处，大模型能力已日趋成熟 — 是时候把二者桥接起来
知识沉淀 AI 赋能 展业提效
▶ 现状与痛点
PAIN 01
知识散落各处
实战经验分散在 群聊 / 邮件 / 文档 / 口头沟通 多个渠道，碎片化严重。同一个问题反复被不同客户问到，每次都要从头组织回答。
PAIN 02
分支客户激增
专业知识未对齐
新客户接入增多、技术水平参差。FIX 协议对接、北上极速接入、算法单配置等问题被反复解答，团队在重复劳动上消耗大量精力。
PAIN 03
衍生品 & 极速业务
专业门槛高
FIX Protocol、算法单、券池、估值、风控术语密集；跨系统对接涉及接入方式、重连机制、延迟优化、日志排查 — 缺结构化知识体系，新人和客户都难快速理解全貌。

Agent 灵魂 ＋ 个人记忆
每次对话前都会现场"装配"的完整 Prompt config/agents/otcclaw.md
静态 灵魂 定义"它是谁、会什么、怎么说话"；动态 {{占位符}} 注入"此刻它记得什么、手上有什么"
▶ 灵魂（静态 Prompt）
❶ 身份 & 能力定位
  • 衍语 / OTC Claw — OTC 衍生品展业助手
  • 客户信息查询与状态流转（Initial Contact ↔ … ↔ PROD，支持 advance / rollback）
  • 交易成交数据查询（按管理人自动展开全部交易对手）
  • 客户数据分析 · 展业建议 · 话术参考
  • 知识库搜索回答 FAQ
  • 工具自举：按需 save_skill 沉淀可复用提示词
❷ 回答风格
  • 简洁专业中文，避免冗长
  • 适度使用 emoji 段落标识（📊 📋 🔍 💡 ✅ ⚠️），不过度堆砌
  • 严禁凭记忆回答数据，必须主动调用工具取最新
  • 建议需结合客户实际状态和需求
❸ 工具使用规范（节选）
  • query_clients 必须从问题里提取 keyword，禁止空参全量查询
  • 文件入库走 import_document，add_knowledge 仅限单条 FAQ
  • 报价 Excel 自动路由：客户报价条款 → import_pricing_schedule；产品利率矩阵 → import_pricing_quote
  • 管理人名 vs Counterparty：query_clients 只识别管理人名，需先 list_customers 取映射
  • "报价"语义消歧：具体客户 → view_client；货币/期限 → query_pricing_quote
  • 需要外部公开信息（行情 / 新闻）→ 浏览器工具链 mcp_devtools_*
▶ 个人记忆（动态注入占位符）
{{permissions}} 权限清单
根据当前用户的 RBAC 角色展开可用工具与数据范围（如销售仅看自己的客户；管理员可见全域）
{{attachments}} 本轮附件
当次消息携带的图片 / Excel / PDF 列表及元信息，供 LLM 判断是否需要调 import_* / parse_excel
{{skills}} Skills 目录
当前 Agent 可用的提示词模板清单（名称 + 简介），命中后通过 run_skill 拉取全文
{{memory}} Agent 维度记忆
跨用户共享的 Agent 长期记忆（SQLite memory 表）：全局 50 条 + 本 Agent 30 条，覆盖业务常识、术语对齐、历史决定。所有用户问同一个 Agent，都看到相同的 memory。
{{user_context}} 用户 × Agent 记忆（Workspace）
千人千面的个人偏好与对话摘要（纯文本 md：data/workspaces/{agent}/{userId}.md）。每次会话结束 LLM 自动摘要追加，下次对话时再注入。记住的是"这个用户在用这个 Agent 时的习惯"。
{{datetime}} 当前时间
服务器本地时间戳，让 LLM 理解"昨天 / 上周 / 本月"等相对时间表达
{{memory}} Agent 维度：跨用户共享，SQLite memory 表
{{user_context}} 用户 × Agent 维度：Workspace md，会话结束自动摘要

▶ 重点关注的异常场景
CRITICAL 多轮工具调用仍未找到答案
Agentic Loop ≥ 3 轮且最终回答带"未找到 / 暂无数据 / 无法确认"等模糊表述。完整保留工具调用链 + 参数 + 返回，进人工复盘队列。
CRITICAL Agentic Loop 打满上限
达到 max_iterations 仍未收敛。可能是工具返回噪声过多、Prompt 规则冲突或模型误解意图。
WARN 知识检索命中 0 条
search_knowledge 返回空 Top-K。多次出现的 query 代表知识盲区，需补录 FAQ 或导入新文档。
WARN 单工具连续失败
同一工具在一轮内重试 ≥ 2 次失败。排查参数提取、RBAC 边界、外部依赖。
WARN P95 长尾耗时
耗时突破 P95 阈值。分段对比定位是 LLM 慢（切轻量模型）还是工具慢（加缓存/索引）。
